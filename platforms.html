JSHint Platforms

Node, Rhino, WSH, etc.

JSHint supports for Rhino, JavaScriptCore and Windows Script Host out of the box!

Rhino

$ make rhino
[...]

$ rhino build/jshint-rhino.js myfile.js

If you don't want to build the jshint-rhino file yourself, you can download it here.

Windows Script Host

> cscript env/wsh.js myfile.js

JavaScriptCore

env/jsc.sh myfile.js

Node

For Node we have a special package that you can install via NPM and then use as an ordinary CLI program:

$ npm install jshint -g
[...]

$ jshint myfile.js
Lint free!

The CLI uses the default options that come with JSHint. Only one extra option is unique to the CLI: globals allows you to define an object of globals that get ignored for every file. Configuration options are defined in a file with JSON content. Example:

{
    "undef": true,
    "unused": true,
    "globals": {
	"MY_GLOBAL": false
    },
    "browser": true,
}

You can specify the configuration line with:

$ jshint --config myconfig.json myfile.js

Use --help to see other available command line options.

Other cool stuff

JSHintr

JSHintr is a web tool that allows you to set your own code standards, easily review a file against these standards, and share the output with other developers.

fixmyjs

fixmyjs is a tool that automatically fixes mistakes—such as missing semicolon, multiple definitions, etc.—reported by JSHint.

Ruby Gem

Another Ruby Gem (jshintrb)

In contrast to the JSHint Ruby gem, this gem doesn't require java.

pre-commit

pre-commit checks your code for errors before you commit it.

Ant Task

Dedicated Ant task to easily automate JSHint in Ant and Maven.

QHint - JSHint in QUnit

Check for errors in your code from within your unit tests. Lint errors result in failed tests.
